The easy Part

“Learning a programming language” is what applies foremost to programmers who already know how to program. For them, picking up a new language is possible within a couple of weeks (and for you too, once you’ve gained some experience), and it does’t matter really that much which one – because the patterns how to accomplish most things are roughly overlapping.

The hard Part

But if you start out, you don’t need to just learn a programming language. First, you want to learn how to program. It means learning how to analyze a problem you want to tackle, finding a solution and express it in code, which is often not intuitive. Plus, you still have to learn a programming language with syntax and semantics, too. So that is two things at once.

Your Reason to learn coding

Do you want to prepair for a career as a software developer? There are different areas where software is developed – the most obvious within the web, is obviously the development of the web itself. But there are other areas like fintech, aircraft systems, robotics, telecom, water supply or internet of things. If you know exactly what you intent to do, you could start with a language of your specific field and go form there. Or maybe better not?

You could also learn a language that’s interesting for you and is particularly suitable for learning, according to those criteria I lay out below. I’m also going to describe the most interesting languages and their idiosyncrasies.

You may want to learn programming, even if you have no plans to become a professional software developer – like me. I’ve worked quite a long time as a designer and in online marketing industry, but have barely written any code at all, since there were always developers working alongside me and tasked with the wizardry.

But I have always been interested in computer stuff, so I’ve dabbled a lot with Linux, server administration and tweaking my configs as a means of procrastination. Real programming was the magical border I haven’t crossed – until recently.

For my part, I want a tool in which I can think, and I’d like to be able to prototype an idea if I have one. Further, I think programming can help us tremendously to develop our minds. I must say that hated mathematics in school and have been really bad at it, but eventually I want to know what is this all about. So I had to start somewhere.

What makes a Programming Language beginner-friendly?

Personality Fit

What kind of animal are you? People weigh values differently and use their cognitive functions in different orders. You may be aware that not everyone thinks in the way you do. Or you may even know you are weird different. Well, it is likely so, and if that weren’t a common phenomenon, mankind would probably not have survived to this day.

Programming languages are created by different minds, within different eras, significantly influenced by fashion. One language may be a natural fit for a certain personality type, and a less fortunate choice for another. That even applies to programming itself – it’s easier for some and harder for others (doesn’t mean those are more stupid).

For our purpose, programming languages can be assigned to the following categories:

  • Pragmatic languages: conservative, efficient, boring.
    “Don’t make me think too much, let’s get this job done”
    (C, Java, Go)
  • Hackish languages: multi-paradigm, anything goes, unopinionated.
    “I pursue absolute freedom and like to explore the problem domain interactively”
    (Perl, JavaScript, Lisp)
  • Constraining languages: single-paradigm, protecting you from yourself, opinionated.
    “I prefer one correct way according a fixed set of rules”
    (Scheme, Python, Haskell, Elm)

Learning Material

For which programming language is the best learning material available? This point is huge! I’d go so far and choose a first language solely on this single criterion.

Most programming books or courses are not made for absolute beginners, but for programmers who want to learn another language. Remember, you want to learn programming, not just a language (You’re going to learn the language on the go, and it will not be your only one).

It’s not that “regular” programming books are “too advanced” to grasp; it’s just that they teach you the wrong things, instead of the things you’ll actually need to learn.

I think it would be wise to prefer material from university profs and/or other reputable personalities to make sure you adopt a well rounded foundation and good style, instead of jumping between random tutorials. The more popular the language, the more crap you’ll run into.

That’s why I recommend a textbook or e-textbook with exercises as your main forge of knowledge, because books are still the most accessible media – you can read a chapter even if you’re queueing up somewhere.

Prepare to dabble with your textbook for quite some time. So you really want an enjoyable one! The most recommended book may not neccesarily be the right one for you; individual taste matters a lot. I’ve listed the best available programming books below with some alternatives, for each language.

Future-proof

Which programming language will be in high demand in late autumn 2026 so I can get filthy rich coding in it? Well, nobody knows and it doesn’t matter. Why? Because it’s about learning to code – which is a timeless skill – not about a particular language (those come and go).

Programming is a skill like being able to ride a bicycle without dying instantly when you fall over. Once you’ve mastered the underlying principle, it will not make much of a difference if you hop on Grandma’s grinder or shred your neighbour’s kid’s tricycle, and it will not matter much if the current trend says in 10 years everyone is riding nothing but mountain bikes with pink tyres.

But! There’s one thing that’s getting more and more important. It’s called concurrency. Most languages in use today have been developed years ago when Moore’s Law was still valid. Back then, everyone expected single processors getting faster and faster, so it was perfectly fine to have languages to make programs running on a single processor, and no one cared to build in concurrency features.

But that is not any more. To keep up with the demands of modern applications, we have no other chance than to put more processors into a computer, or to run the application distributed on multiple computers. To leverage the power of multi-core processors, we need to be able to program our software conveniently in a way to utilize all available cores.

It leads to two things:

  1. Using a programming language that is designed with concurrency in mind and simplifies building concurrent software
  2. Learning how to program concurrent software

Community and Popularity

Popular languages have some benefits – for instance bigger ecosystems, which leads to many things built around the language to make the life of (experienced) programmers easier.

Libraries are part of the ecosystem: those are pieces of code written by other programmers that can be re-used, so a developer has less work and doesn’t need to code everything from scratch.

Obviously, popular languages have also bigger communities. You don’t need a huge community, but it should be a welcoming one. Smaller communities have their benefits, too.

Expressive Power and Abstraction

Expressivity means how simple it is to say certain things. There are a couple of factors who determine how expressive a programming language can be:

Low-level or High-level?

Low-level does not mean “worse” or “inferior” than high-level – it just refers to the fact that computers are built in many layers on top of each other, and the levels close to the physical machine are called the “low level”. You could also call it “foundation”, if you prefer a less judgementally sounding term xP

High-level languages tend to be more expressive than low-level languages, because with each layer up, the grade of abstraction increases and more powerful ways to express complex things come into play.

How simple is it to build Abstractions?

What is abstraction? It means hinding the foundational workings of the level(s) below (eg. the physical machine) so you don’t have to deal with them directly and can focus on your problem/idea instead.

An analogy: A console used to control the lightshow of a concert is an abstraction over fumbling with the electricity cables and tieing them together to make light bulbs glow in a certain order. Now think about doing the light show only by messing with the wires. This is dangerous and way harder (nearly impossible) than using the console. And so it is with programming, too.

Is it a wordy or concise Language?

Do you have to write a lot of boilerplate or repeat yourself? In most cases you want an expressive high-level language; not overly verbose but also not too concise, so there’s just the right amount of redundancy left to keep it comprehensible.

Side-read: programming languages ranked by their expressiveness

Syntax and Consistency

As a novice coder, you probably want to start out quickly, and you’ll need enough brain capacity left to maintain focus on the important things, without the language getting in your way. So you want a simple syntax that’s easy to pick up and remember.

A thoroughly designed programming language suits well for learning, because you will encounter less surprises and pitfalls. Consistency helps you to derive further things from things you already have learned before.

Maturity

While it may be tempting to start with the current hyped language everyone is raving about, a mature programming language has the benefit of an organically grown user base, flaws ironed out, stuff being written about it, better tooling, etc.

A mature language doesn’t change every couple of months, so you can be sure to learn the “right” things – you don’t have to worry if you’re learning old stuff without knowing.

Which language has been around for quite a while, but implements modern, interesting and useful concepts that may be relevant in the future?

Tooling and Usability

Your language of choice should come with tooling that’s easy to use. It includes things like editor support (syntax highlighting, “jump-to-definition”, looking up documentation), helpful (or at least comprehensible) error messages, and package management for libraries.

A good selection of libraries is desirable for industrial use, but doesn’t matter sooo much in the beginning. Some languages can even use libraries from other programming languages / ecosystems, and go even further than that.

Consider to pick a language that comes with a REPL, sometimes called console or toplevel. That’s basically like the command line running in a terminal emulator, but instead of shell commands you can run single pieces of your code. Well, nowadays many languages come with a REPL. But some go even further than that and enable true interactive development.

The best Programming Language to learn Coding

Here is a carefully picked selection of programming languages well suitable to learn programming. Some of them may fit better for you, and some a bit less:

ANSI C

Welcome to the low-level. You may hear here and there that C is the foundation for everything and everyone must learn programming in C before even thinking about anything else – but no, don’t waste your time: Learn C if you have a genuine interest in systems programming or another reason to work very close to the machine.

If that is the case, you may also look into the Arduino electronics platform, coming with the Arduino programming language (which is basically C).

Programs written in C tend to be much more performant than programs written in other languages. In other words, it is the fastest (practical) programming language. Performance-critical algorithms and other parts of applications are usually written in C.

Don’t confuse it with C++ (an extension C, but huuuge) or C# (a Microsoft thing). Look for “ANSI C” – the standard.

  • Get startet with C or Arduino
  • C programming book for beginners:
    • The C Programming Language, 2nd Edition (paper/Kindle)
    • Beginning C for Arduino, Second Edition: Learn C Programming for the Arduino, 2nd Edition (paper/Kindle)

Lua

On the surface, Lua is quite similar to JavaScript – just without the ugly parts. Lua is minimalist, but fast and powerful, extendable and comprehensible, well designed and consistent. Therefore it is super easy to learn for Muggles Non-programmers.

The language is mainly used in the game industry (e. g. World of Warcraft, Roblox). Lua goes very well with C and it can be embedded into C code, so that high-level program logic – like character behavior – can be scripted in Lua, while the other parts of the software in need for raw metal speed can be written in C.

Lua can be used everywhere and it’s quite simple to get started and build something with it: You might want to make your own 2D game with the Löve game engine, you can use it to code native games and other apps for Android.

There’s a really nice IDE (“Integrated Development Environment”) for Lua programming, called “ZeroBrane Studio”; think of it as an extended editor especially for Lua.

As far as I know, there are not many textbooks available, but I’ve linked the standard book written by the language inventor down below, and there’s a wiki and many many user tutorials.

  • Get started with Lua and the ZeroBrane Studio
  • Lua programming books for beginners:
    • “Programming in Lua 1st Edition” (free online)
    • “Programming in Lua 4th Edition” (e-book) | Paper book

Clojure

As a total noob, I’ve been looking for a language with a super simple syntax, because learning syntax scared me the most. Clojure is a member of the Lisp family, those have the simplest syntax of all programming languages. Further, it seems to be the most expressive general purpose language, and it’s designed especially with concurrency in mind.

The language is built on top of the Java platform. Software built on Java can run on almost any major operating system – Windows, Linux, Mac, etc. Clojure can be used for different things, even for artsy stuff. Just as an example: you can do live coding and make algorithmic music, and there are things like algo-raves (yup, that exists …).

It’s also a practical language to build serious business software with, because it can use Java’s huge library ecosystem. And there’s ClojureScript, a dialect of Clojure that compiles to JavaScript (so you you won’t have to write web apps in JavaScript). Clojure Developers reap the biggest salaries according to several studies. Not that it means something, just sayin’.

I recommend the book “Clojure for the Brave and True”; it had given me a few enlightening moments: lively visions of how all the parts interlock: functions built of functions, taking functions and spitting out functions, like moving parts of a giant machine. The book convinced me also to take a look at Emacs, which is a mindblowing beast of itself.

Clojure’s drawbacks are those not-so-helpful error messages. Plus, there’s that Java underbelly who sticks out everywhere, coming with it’s own terminology.

  • Get started with Clojure
  • Clojure programming book for beginners:
    • “Clojure for the Brave and True” (paper/Kindle)

Scheme and Racket

Pleasantly surprised by Clojure, I’ve looked up other languages from the Lisp family. Scheme is usually recommended to beginners who are serious about computer science. It’s quite opinionated, but it is also minimalist and therefore good for learning.

Racket, is an extended Scheme-like language (it has been called PLT Scheme before) that emphasizes “language oriented programming”. It means you design your own small (domain specific) programming languages for your application and then write your application in that language. But you don’t have to – you can just use “normal” Racket. So don’t bother with that for now.

Racket is also a platform for research and to create and run different programming languages. Scheme can run on Racket, so even if you just want to learn the Scheme language, you can do it comfortably in Racket. That may sound a bit confusing, but it actually isn’t. You’ll see.

The most recommended book on programming is the timeless classic “Structure and Interpretation of Programming Languages”, abbreviatet “SICP”. It was written as an introduction for computer science students at MIT, using Scheme. But for me it was too much as a very first book on programming. But check it out anyway, it’s really philosophical.

If that’s too heavy for you, pick one of the others. I’ve also started to read “How to Design Programs”, which is supposedly easier, but found it super earnest and dry. A stripped down teaching language (think: training wheels) is used within that book, and I’m no friend of this approach. And the other books are quite different.

You can get started very easily with Racket and Scheme – one single install and go, no matter if you are on Windows, Linux or Mac. You don’t even have to be familiar with the command line, because it comes with DrRacket, an easy to use programming environment (IDE).

  • Get started with Scheme or Racket and the DrRacket IDE
  • Scheme and Racket programming books for beginners:
    • “Realm of Racket: Learn to Program, One Game at a Time!” (paper/Kindle)
    • “How to Design Programs: An Introduction to Programming and Computing” (paper/Kindle) | Free online
    • The little Schemer (Paper book)
  • Introduction into computer science:
    • “Simply Scheme” (free online) | Paper book
    • “Structure and Interpretation of Programming Languages” (free online) | Paper book

Common Lisp

The first reason for me to use a programming language wasn’t coding, actually I wanted a better calculator. A REPL makes a way better calculator than the silly concept of an interface resembling a physical pocket calculator with big buttons and a single line display.

Well, here Lisp’s prefix notation felt really weird for doing arithmetik: Instead of 10 + 3 + 4 + 6 + 7 you write (+ 10 3 4 6 7). All Lisp languages have this uniform and simple syntax. It makes it easier to write programs who write programs who … well, that’s for later. The syntax is easy to remember so it doesn’t get in your way, and you can concentrate on the things that really matter.

I think the most compelling books and essays have been written on Lisp, and it has the best learning material I could find. Well, Lisp has been around for a while.

What really fascinates me, is that Lisp is no “black box”, where you write some keywords and then something underneath does some things and you don’t know what and why. Lisp is different: It is built on a few simple pieces, and everything else is built from those pieces – it is transparent and easy to paint a mental picture how it actually works.

The tooling is excellent and designed for interactive development. There’s a really nice App for iOS and Android called “CL REPL”. You can use it to tinker around when ever you feel like “Damn, I wish I could write some Lisp right now!” You can. It’s in fact a complete Common Lisp IDE with SLIME, the “Superior Lisp Interaction Mode for Emacs”.

But usually you want it on your laptop. I recommend Emacs with SLIME, and it’s easy to install: use Portacle, which has everything bundled together with batteries included. If you don’t know Emacs yet, do:

  1. Menu -> “Options” -> “Use CUA keys”
  2. Menu -> “Options” -> “Save options”

– then copy & paste works like everywhere else.

  • Get started with Common Lisp and the Protacle IDE
  • Lisp programming books for beginners:
    • “Common Lisp: A Gentle Introduction to Symbolic Computation” (paper/Kindle) | Free PDF
    • “Land of Lisp” (paper/Kindle)

Elixir

It’s a young language, built on top of the Erlang platform, a solid and integral building block of today’s telecommunication networks. Your mobile provider’s infrastructure runs likely on Erlang, and so does WhatsApp.

Elixir has an approachable syntax and leverages an interesting concept, Erlang’s actor model. It makes Erlang/Elixir specifically suitable for concurrency, and it’s a natural fit for distributed computing, peer-to-peer networking, messaging apps and servers with hundreds of thousands of simultaneous active users, internet of things, etc.

Some things may be a bit confusing for noobs: For instance, you can define functions in 3-5 (?) different ways: there are long forms, and there are short forms, and not all of those work everywhere, but which one does, depends on the context. Wtf?

Nevertheless, it’s quite easy to get started with Elixir. You won’t build anything graphical with Elixir directly, it’s a language for servers, networking and everything that has to do with communication. How about building a chat bot?

  • Get started with Elixir
  • Elixir programming book for beginners:
    • “Introducing Elixir: Getting Started in Functional Programming” (paper/Kindle)

Haskell

Elm pointed me to Haskell, where most of Elm’s ideas come from. So if I like Elm and it’s a browser-only language, why not learn “the real thing”?

Haskell has a simple syntax, it’s just uncommon for programmers that are ingrained with the syntax of C/Java/PHP/JavaScript. However, Haskell’s syntax relies a lot on cryptic infix operators (also custom ones) and with it come things like the precendece of those operators (from 0-9).

Contrary to popular belief, basic Haskell is easy to learn; especially as a person who has never programmed before. It may be less easy for experienced programmers because of all the preconceptions they have about programming. But it’s not a compact language, so it will get complicated later when you’ll want to work with libraries and encounter code others have written. Haskellers tend to be clever, and they like to use the advanced features the language and its extensions have to offer.

Unfortunately, it’s not so straightforward to get started with Haskell … well, it depends. Downloading the compiler and starting ghci (the REPL) to evaluate some expressions is no big deal, and to compiling some code neither. But if you go a bit further and want some more convenience and install libraries, there are a couple of different approaches which tooling set to use, and they overlap in functionality, and … I think that’s a bit confusing.

Not all hope is lost: You could also start with Elm, and I think the static type system is easier to grasp with Elm. You can still move on to Haskell later, if you feel like. But if you wanna give it a shot, download Stack and then install everything else via Stack.

  • Get started with Haskell (Stack)
  • Haskell programming books for beginners:
    • “Haskell - the craft of functional programming” (paper/Kindle)
    • “Haskell Programming from first Priniples” (sample)

JavaScript

With JavaScript, you can build all kind of things in the browser and show it to your internet friends, without having them to download or install anything (that’s “easy deployment” of its finest). JavaScript is the only language browsers support directly, still – and that’s why everybody and their dog’s fleas are coding JavaScript.

The language has good tooling and is easy to start with, and you may have heard “all you need is a web browser and a text editor”. Well, um, no, unfortunately, that’s not really the truth.

You’ll need willingness to suffer. The language has a ton of WTFs and ugly corners, and you’ll need to learn to avoid and remember those pitfalls, and using a linter is mandatory (that’s a tool that checks your code and tells you where the problematic parts are). Expect to write a lot of code to work around the poor language design (meh!).

Overall, syntax and semantics result in a verbose frankenstein language that’s a pain to read. Of course there are the usual suspects who disagree and even “love” JavaScript. These are the people who either don’t know any better, or who suffer from a variant of the Stockholm Syndrome. Honestly, that doesn’t make it a good choice for a first time programmer, but despite the fact, many start to learn programming with JavaScript, unfortunally.

It is an imperative language at the very heart, but you can write object-oriented and functional style too. JavaScript is far from standing still, but assimilating new features with each version and therefore getting bigger and bigger. Resistance is futile. Or is it?

If you just wanna build web apps or browser games, you do have other choices: check out Clojure(Script), or Elm down below.

  • Get started with JavaScript
  • JavaScript programming book for beginners:
    • “Eloquent JavaScript” (paper/Kindle) | Free online

Elm

Elm is a language that compiles to JavaScript: You write your website’s code in Elm, and then the Elm compiler writes the JavaScript for you, so that your web browser can run it.

If that sounds like it makes no sense, think about it this way: All programming languages compile down to another language on a lower level anyway, and those again and again, until your code’s representation is only 11010101. Elm is just another layer on top.

Why not writing JavaScript directly? Elm comes with a few interesting features: first and foremost a static type system (like Haskell), which helps you to be (more) sure about your code is correct (think: less undiscovered bugs). And you don’t have to deal with JavaScripts screwed type coercion. Secondly, Elm supports the functional programming paradigm: That makes it’s easier to reason about what code actually does. And it comes with immmutable values (Ever writhed in disgust when you saw x = x + 1? Then this is for you).

The Elm syntax is simple and the Elm compiler famous for its helpful error messages – like a built-in teacher who pushes you gently into the right direction on your path to statically typed functional programming. And you will learn one way how to structure your application: “The Elm Architecture”.

That’s why I consider Elm very well suited for learning, despite the fact that it’s still a very young and domain-specifig language that runs exclusively in the browser. There is not much literature out there, but Elm opens the door to Haskell.

Elm is designed as a cathedral – the installation is easy, tooling on par. But the language is quite opinionated: it’s encouraged to do certain things like intended by the creators, similar to Python. But that has benefits too, and it’s one of the reasons why everything within the Elm ecosystem fits so well together.

  • Get startet with Elm
  • Elm programming book for beginners:
    • “Beginning Elm” (free online)

Python

I’ve considered learning Python a couple of years ago, but after some research I found other languages more elegant and interesting, and it seems that Python is a lot of lackduster – looking elegant on the surface, but inconsistent when you look under the rug.

I’m listing it here nonetheless, because it’s one of the 3 most popular languages right now (behind C and Java). Python emphasizes user friendliness and it has really comprehensible error messages. The language is used in many domains, from small scripts to scientific computing. Python usually serves as the glue code to connect all the parts of a particular software project.

There are Python libraries for everything, which adds to popularity. On the technical site, many of those libraries are actually written in C for performance reasons, because Python itself is by far the slowest of the languages listed here. If you want a scripting language which is even easier to learn but really fast, you may have some fun with Lua.

Python is also a very opinionated language: The creators encourage that certain things should be done in “the right way” – which is called ’pythonic’. The opposite is ’unpythonic’: To describe something as ’clever’ is not considered a compliment in the Python cult-ure. Well, you may be more pragmatic and don’t care, but that mindset is putting me off.

  • Get started with Python
  • Python programming book for beginners:
    • Automate the Boring Stuff with Python (paper/Kindle)

Julia

Julia is developed as a high performance vehicle for technical and scientific computing, like data science, simulations, as an alternative to Python and a couple other languages in the field. Most scientists already how to program their stuff, so don’t expect much introduction material for total beginners.

It’s still a young language, but pushed by a couple of organizations. So there’s more of an ecosystem than you would expect for a language in that early state.

I’ve considered to learn Julia because there are some interesting aspects. Julia can do some things that Lisps can, but contrary to Lisps, Julia neither uses prefix-notation syntax nor nestet parantheses.

Julia looks friendly and easy to start with, and you should definitely check it out if you’re a student or scientist. I think it’s a very promising general-purpose language.

  • Get startet with Julia
  • Julia programming book for beginners:
    • “Think Julia: How to Think Like a Computer Scientist” (paper/Kindle) | Free online

Conclusion

Choose well, but don’t obsess too much about it: I’ve already done that part for you! You are going to learn a couple more programming languages after your first one, so it’s not a lifetime decision, even if you feel like that now.

Some courses and books teach programming using stripped-down teaching languages, or even pseudocode. I think it’s not a good idea, because you cannot use the teaching language to build anything useful along the way; you learn something you have no use for later on; and the teaching languages are often not subsets of a real programming language, which may lead to confusion thanks to a slightly off terminology.

Mkay, but which one now … ?

After all, I think Racket ticks most boxes. The language design and usability are awesome, the syntax clear, it has a fantastic documentation, and Racket can teach you all the interesting concepts. No wonder, it’s actively used for computer science and programming language research. It’s also a bit academic. Oh, I said the A-word xP

The second one is Lua. Here, the same is to say about syntax, language design and documentation. It may not teach you all those fancy concepts like Racket can, but Lua is minimalist, so you’ll be able to fit the whole language in your head, which is quite impossible with the other languages. Lua is very practical; you can use it nearly everywhere.

My personal choice

On my shortlist there were Racket, Lua, Haskell and Common Lisp. Ultimately, I started with Common Lisp. So what’s my motivation for that?

I’ve looked into many books, and it turned out that I have a favorite textbook on programming and I want to work with that one first. It is great. Many many exercises. Let’s see if I’ll stick to Common Lisp afterwards :)

Secondly, I favour convenience and exploration. The interactive Common Lisp development workflow is far more advanced than anything I’ve seen.

Lisp goes very well with Emacs. Important to say is, you don’t have to use Emacs to code in Common Lisp. There are plugins for Atom, VSCode and others.

I was already fascinated by Emacs since my first encounter with Clojure. Emacs is written in Emacs Lisp, another programming language within the Lisp family. Emacs Lisp and Common Lisp are similar enough (on a trivial level) to avoid confusion while switching back and forth beetween both <- Compare the syntax of the four most used Lisps. In fact, I’m writing more Emacs Lisp than Common Lisp at the moment. But that’s a story for another time.

Although Racket, Lua, Haskell and Common Lisp are not the most used languages in the industry, they work well to learn programming. After learning one of them, you could go on and make a popular 1st-tier bread-and-butter-language your second endeavour, like JavaScript, Python, Java, Scala, Go or C++.
